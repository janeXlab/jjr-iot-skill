# 设备服务调用流程指南

> 📖 如何正确调用物联网设备服务（如：拍照、重启等）

---

## 📋 正确流程

### 步骤 1：查询设备服务列表

**接口：** `GET /iot/open/iot/device/serve`

**用途：** 查询设备支持的所有服务及其参数

**请求：**
```bash
curl --location --request GET \
  'https://gateway.jjr.vip/iot/open/iot/device/serve?productKey={productKey}&deviceName={deviceName}' \
  --header "Authorization: Bearer {TOKEN}"
```

**返回示例：**
```json
{
  "code": 200,
  "message": "查询成功",
  "result": [
    {
      "identifier": "takePhoto",
      "name": "拍照",
      "inputData": [
        {
          "identifier": "resolution",
          "name": "分辨率",
          "dataType": "string",
          "required": false,
          "enum": ["1920x1080", "1280x720", "640x480"]
        },
        {
          "identifier": "quality",
          "name": "图片质量",
          "dataType": "int",
          "required": false,
          "min": 1,
          "max": 100
        }
      ],
      "outputData": [
        {
          "identifier": "imageUrl",
          "name": "图片 URL",
          "dataType": "string"
        }
      ]
    },
    {
      "identifier": "reboot",
      "name": "重启设备",
      "inputData": [],
      "outputData": []
    }
  ]
}
```

**关键信息：**
- `identifier`: 服务标识符（调用时必需）
- `inputData`: 输入参数列表（调用时需要传入的参数）
  - `identifier`: 参数标识符
  - `dataType`: 参数类型（string/int/float/boolean）
  - `required`: 是否必填
  - `enum/min/max`: 参数约束

---

### 步骤 2：调用设备服务

**接口：** `POST /iot/open/iot/device/serveSet`

**用途：** 执行设备服务

**请求：**
```bash
curl --location --request POST \
  'https://gateway.jjr.vip/iot/open/iot/device/serveSet?productKey={productKey}&deviceName={deviceName}&identifier={identifier}&input={inputJSON}' \
  --header "Authorization: Bearer {TOKEN}"
```

**参数说明：**
- `productKey`: 产品 Key
- `deviceName`: 设备名称
- `identifier`: 服务标识符（从步骤 1 获取）
- `input`: JSON 格式的输入参数（需要 URL 编码）

**示例：调用拍照服务**
```bash
# 假设拍照服务需要 resolution 和 quality 参数
input='{"resolution":"1920x1080","quality":90}'
input_encoded=$(python3 -c "import urllib.parse; print(urllib.parse.quote('$input'))")

curl --location --request POST \
  "https://gateway.jjr.vip/iot/open/iot/device/serveSet?productKey=xxx&deviceName=xxx&identifier=takePhoto&input=$input_encoded" \
  --header "Authorization: Bearer {TOKEN}"
```

**返回示例：**
```json
{
  "code": 200,
  "message": "调用成功",
  "result": {
    "imageUrl": "https://xxx.com/photo.jpg",
    "timestamp": "2026-04-16 15:30:00"
  }
}
```

## 🛠️ 完整示例：产业园 5 拍照

### 步骤 1：查询服务列表

```bash
# 获取 Token
TOKEN=$(python3 -c "
import json, requests
with open('config.json') as f:
    c = json.load(f)
r = requests.post('https://gateway.jjr.vip/iot/oauth2/token',
    data={'grant_type': 'client_credentials',
          'client_id': c['client_id'],
          'client_secret': c['client_secret'],
          'scope': 'open'})
print(r.json()['result']['accessToken'])
")

# 查询服务列表
curl -s --location --request GET \
  'https://gateway.jjr.vip/iot/open/iot/device/serve?productKey=YOUR_PRODUCT_KEY&deviceName=YOUR_DEVICE_NAME' \
  --header "Authorization: Bearer $TOKEN" | jq .
```

### 步骤 2：根据返回结果调用服务

**情况 A：设备支持拍照服务**
```bash
# 如果返回中有 takePhoto 服务
identifier="takePhoto"
input='{}'  # 或根据 inputData 填写参数
input_encoded=$(python3 -c "import urllib.parse; print(urllib.parse.quote('$input'))")

curl -s --location --request POST \
  "https://gateway.jjr.vip/iot/open/iot/device/serveSet?productKey=YOUR_PRODUCT_KEY&deviceName=YOUR_DEVICE_NAME&identifier=$identifier&input=$input_encoded" \
  --header "Authorization: Bearer $TOKEN"
```

**情况 B：设备不支持拍照服务**
```
返回：{"code": 300, "message": "服务不存在"}
```

**解决方案：**
- 设备可能只支持查询属性（如 `imgUrl`）
- 使用属性查询接口获取最新图片
- 或更换支持拍照的设备

## 📝 注意事项

### 1. 服务标识符必须准确
- 从步骤 1 的返回结果中获取 `identifier`
- 不要自己猜测（如 `takePhoto`, `camera`, `capture` 等）
- 不同设备的标识符可能不同

### 2. 输入参数格式必须正确
- `input` 必须是合法的 JSON
- 必须进行 URL 编码
- 参数名称必须与 `inputData` 中的 `identifier` 一致
- 参数类型必须匹配（string/int/float/boolean）

### 3. 必填参数不能遗漏
- 检查 `inputData` 中 `required: true` 的参数
- 可选参数可以不传或使用默认值

### 4. 参数值必须在约束范围内
- `enum`: 只能是枚举值之一
- `min/max`: 必须在范围内
- `len`: 字符串长度限制

## ⚠️ 常见问题

### Q1: 返回"服务不存在"
**原因：**
- `identifier` 填写错误
- 设备不支持该服务

**解决：**
- 重新执行步骤 1，确认正确的 `identifier`
- 查询设备属性，看是否有替代方案

### Q2: 返回"参数错误"
**原因：**
- `input` JSON 格式错误
- 缺少必填参数
- 参数类型不匹配

**解决：**
- 检查 JSON 格式
- 对照 `inputData` 检查必填参数
- 确认参数类型（string 要加引号）

### Q3: 返回 500 错误
**原因：**
- 服务器临时故障
- 网络问题

**解决：**
- 等待几秒后重试
- 检查 Token 是否过期
